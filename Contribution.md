# Contributing Guidelines

Thank you for your interest in contributing to this project!  This repository aims to provide a clean, reproducible simulation framework for
evaluating delay-aware control strategies (SAB-PC) under blockchain-style consensus latency.

## How to Contribute

### 1. Reporting Issues
If you encounter a bug, unclear documentation, or unexpected behavior:
- Open an Issue on GitHub
- Include a minimal reproducible example when possible
- Describe expected vs. actual behavior

### 2. Submitting Pull Requests
Pull Requests (PRs) are welcome for:
- Code improvements or refactoring
- Additional delay models or MJLS modes
- New controllers (e.g., MPC, robust control, adaptive control)
- Real blockchain delay traces
- Additional plotting or analysis tools
- Documentation improvements

Thank you for helping improve this project!
